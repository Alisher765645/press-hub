# PRD: PRESS HUB — Единый центр пресс-службы нефтегазовой компании

**Версия:** 1.0  
**Дата:** Июнь 2026  
**Статус:** Прототип реализован  
**Владелец продукта:** Пресс-секретарь / Руководитель пресс-службы

---

## 1. Обзор продукта

PRESS HUB — внутренняя веб-платформа для централизованного управления коммуникационной деятельностью нефтегазовой компании и её 24 дочерних предприятий. Платформа объединяет планирование контента, хранение материалов и ИИ-обработку в едином интерфейсе.

### Проблема

Пресс-служба головной компании координирует 24 дочерних предприятия. Без единого инструмента возникают:
- Дублирование тем и конфликты публикаций между предприятиями
- Отсутствие видимости предстоящих задач у ответственных
- Разрозненное хранение пресс-релизов и аналитики
- Высокие трудозатраты на ручную обработку больших объёмов текстов

### Решение

Единая платформа с тремя функциональными блоками: контент-календарь, архив материалов и ИИ-ассистент — с персистентным хранилищем данных между сессиями.

---

## 2. Пользователи

| Роль | Описание | Основные задачи |
|---|---|---|
| Пресс-секретарь (главный) | Владелец продукта, координатор | Планирование, контроль, итоговая публикация |
| Пресс-атташе дочернего предприятия | Исполнитель по своему предприятию | Просмотр своих задач, загрузка материалов |
| Аналитик | Подготовка отчётов и аналитики | Работа с ИИ-ассистентом, архивом |
| Руководитель | Контроль активности | Просмотр статистики и контент-плана |

---

## 3. Функциональные требования

### 3.1 Контент-план (Календарь)

**Описание:** Интерактивный месячный календарь для планирования всех коммуникационных задач по 24 предприятиям.

**Требования:**

- Отображение событий в сетке по дням недели (Пн–Вс)
- Навигация по месяцам (назад / вперёд)
- Добавление события через клик по дате или кнопку «+ Добавить событие»
- Поля события: название, дата, тип материала, ответственное предприятие, описание/ТЗ, приоритет
- Фильтрация по предприятию и типу материала
- Переключение видимости предприятий через боковую панель
- Боковой виджет «Предстоящие события» — ближайшие 8 событий по дате
- Цветовая кодировка по типу материала (см. раздел 5)
- Индикатор высокого приоритета (🔴)
- Текущая дата выделена визуально

**Типы материалов:**

| Тип | Код | Цвет |
|---|---|---|
| Пресс-релиз | `pr` | Синий |
| Отчёт / Аналитика | `report` | Зелёный |
| Медиазапрос | `media` | Оранжевый |
| Соцсети | `social` | Фиолетовый |
| Брифинг / Конференция | `briefing` | Красный |

**Приоритеты:** Высокий, Обычный, Низкий.

---

### 3.2 Архив материалов

**Описание:** Единое хранилище всех созданных и загруженных материалов пресс-службы.

**Требования:**

- Карточное отображение материалов
- Поиск по заголовку, тексту, названию предприятия
- Фильтрация по типу материала и предприятию
- Просмотр полного текста материала в модальном окне
- Добавление нового материала вручную: заголовок, тип, дата, предприятие, текст, теги
- Поддержка тегов (через запятую) для каждого материала
- Кнопка «Отправить в ИИ» — мгновенная передача текста в ИИ-ассистент
- Автосохранение результатов ИИ-обработки в архив
- Данные сохраняются между сессиями (персистентное хранилище)

---

### 3.3 ИИ-Ассистент

**Описание:** Встроенный ИИ на базе Claude для обработки больших объёмов текстов и генерации визуальных материалов.

**Режимы работы:**

| Режим | Описание |
|---|---|
| Выжимка | Структурированные ключевые факты, цифры, необходимые действия |
| Аналитика | Тренды, риски, возможности, коммуникационная оценка |
| Пресс-релиз | Генерация готового пресс-релиза с заголовком, лидом, цитатой руководителя |
| Контент-план | Предложение плана на месяц с датами, типами и ответственными |
| Инфографика | Извлечение 5–7 ключевых показателей и отрисовка визуальной карточки |

**Ввод данных:**

- Текстовое поле для вставки контента
- Загрузка файлов: TXT, DOCX, PDF, CSV (несколько файлов одновременно)
- Прогресс-бар при загрузке файлов
- Контекст предприятия (опциональный выбор)

**Вывод:**

- Текстовый результат с возможностью копирования в буфер
- Сохранение результата в архив одной кнопкой
- Для режима «Инфографика»: canvas-визуализация с возможностью скачивания PNG

**Инфографика (технические детали):**

- До 6 показателей в сетке 3×2
- Тёмный фон, цветовые акценты по типу показателя
- Скачивание в формате PNG

---

### 3.4 Сводная статистика

На странице контент-плана отображаются 4 счётчика:

- Всего событий в контент-плане
- Событий в текущем месяце
- Материалов в архиве
- Количество подключённых предприятий (24)

## 4. Нефункциональные требования

### 4.1 Хранение данных

- Все события контент-плана и архивные материалы сохраняются в персистентном хранилище (Storage API)
- Данные доступны при следующем открытии без повторного ввода
- Формат хранения: JSON

### 4.2 Интерфейс

- Тёмная тема (корпоративный стиль: чёрный фон, акцент — нефтяной янтарь #E8A020)
- Типографика: Inter (основной), Bebas Neue (заголовки)
- Адаптивная вёрстка: десктоп (≥900px) и мобильный (<900px)
- Тост-уведомления о действиях пользователя
- Модальные окна с закрытием по клику на фон

### 4.3 Производительность

- Мгновенный отклик интерфейса при фильтрации и навигации по календарю
- Запросы к ИИ API — асинхронные, с индикатором загрузки

### 4.4 Безопасность

- Приложение работает в браузере без серверной части
- API-ключ Anthropic передаётся через прокси (не хранится в коде)

## 6. Структура навигации

```
PRESS HUB
├── Контент-план (Календарь)
│   ├── Статистика (4 счётчика)
│   ├── Календарная сетка
│   │   └── Модальное окно добавления события
│   └── Сайдбар
│       ├── Предстоящие события
│       ├── Список предприятий (переключение)
│       └── Легенда типов
├── Архив материалов
│   ├── Поиск + фильтры
│   ├── Карточки материалов
│   ├── Модальное окно просмотра материала
│   └── Модальное окно добавления материала
└── ИИ-Ассистент
    ├── Панель ввода
    │   ├── Выбор режима (5 режимов)
    │   ├── Загрузка файлов
    │   └── Текстовое поле

        └── Canvas-инфографика (режим «Инфографика»)
```

## 7. Пользовательские сценарии

### Сценарий 1: Планирование месяца
1. Пресс-секретарь открывает вкладку «Контент-план»
2. Переходит на нужный месяц
3. Кликает по дате и добавляет событие с указанием ответственного предприятия
4. Предприятие видит свои задачи в календаре с фильтром по компании

### Сценарий 2: Обработка большого текста
1. Пресс-атташе получает объёмный отчёт (10+ страниц)
2. Открывает вкладку «ИИ-Ассистент», выбирает режим «Выжимка»
3. Загружает файл или вставляет текст
4. Получает структурированную выжимку за секунды
5. Сохраняет результат в архив

### Сценарий 3: Генерация инфографики
1. Пресс-секретарь вставляет данные производственного отчёта
2. Выбирает режим «Инфографика»
3. ИИ извлекает ключевые показатели и генерирует визуальную карточку
4. Скачивает PNG для публикации

### Сценарий 4: Поиск по архиву
1. Журналист запрашивает историческую информацию
2. Пресс-атташе открывает «Архив материалов»
3. Вводит ключевое слово в поиск (например, «добыча Q1»)
4. Находит нужный пресс-релиз и просматривает полный текст

## 8. Ограничения текущего прототипа

| Ограничение | Описание | Приоритет доработки |
|---|---|---|
| Нет авторизации | Все пользователи видят одни данные | Высокий |
| Нет ролей доступа | Предприятие видит все 24 компании | Высокий |
| Нет уведомлений | Нет рассылки о предстоящих задачах | Средний |
| Нет экспорта PDF/Excel | Только скачивание инфографики PNG | Средний |
| Storage API — браузерный | Данные не синхронизируются между устройствами | Высокий |
| Нет комментариев к событиям | Нет обсуждения задач внутри системы | Низкий |
